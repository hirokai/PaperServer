About

Recent progress in IT in the field of scientific publication allowed us to access vast amount of papers in a short amount of time. Yet, the efficiency in literature search can still be improved by using IT even more effectively to manage all those papers.

There are many literature management softwares, but most of them rely on managing pdf files of papers. PDF is not necessarily great platform for reading papers on the computer screen (read this).

Recently, more and more online journals try to implement rich browser UI on their websites and try to make the navigation in the paper easier. However, those UIs are often too rich and slow. Also, you need to operate differently for different publishers.

PaperServer was developed to solve all those problems.

Unified UI for all online journals by the analysis of HTML

PaperServer analyzes and "understands" the HTML file of an online journal paper. This makes it possible to layout the paper into a uniform UI. Not too much decoration, and fast browsing by keyboard shortcuts. Everything is done in a web browser, and you can use all features just by installing a browser plugin (currenly only Google Chrome plugin).

All papers you read are uploaded from the web browser to PaperServer automatically

You no longer have to keep a memo of your literature search.

This schematic shows how PaperServer works. All data uploaded to PaperServer are kept private, and never get shared.

Better mobile browsing

Virtually no way exists for efficiently and comfortably read a paper on the screen of mobile environment so far. PaperServer can generate EPUB from the analyzed paper data. You can read a generated EPUB by a mobile EPUB reader such as Stanza.

Advanced history search and analysis of the library (not implemented, and planned)

This will make your "manual" management of papers (such as tagging, organizing into folders) unnecessary.

No SNS feature

I think SNS feature is not useful in this kind of software, and that may be even distracting.

Free, open source software

By making the source code of a software open to everyone, anyone can inspect the software and this improves security and validness of the software. You may also run PaperServer on your own server in your lab.（It was tested on Mac OS X 10.8 and Ubuntu 12.04）。The entire source code is distribtued at Github under GPL license. It was developed in Haskell with Yesod web server framework. I also look for a collaborator, and email me if you are interested.

Major features

Already implemented

  • Browsing papers in a web browser with a unified layout and UI.
  • Simplified, text-based data makes the loading of the page faster than the original online journal websites.
  • You can do most operations by keyboard shortcuts.
  • All papers are saved automatically.
  • EPUB generation from the paper data.
  • Citation export in RIS.
  • Tagging of papers
  • Free. Open source software.

Planned

In the order of priority:
  • Cover more journals.
  • Support mobile browsing environment.
  • Advanced search features.
  • Table of contents.
  • Library import from other softwares.
  • Customized UI.
  • Advanced search of history and activities.
